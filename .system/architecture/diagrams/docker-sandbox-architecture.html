Security Hardening
🔒

Read-Only Root

Root filesystem is read-only. Writes only to tmpfs (/tmp, /var/tmp, /run) and bind mounts.

⛔

Cap Drop ALL

All Linux capabilities dropped. No privilege escalation via --security-opt no-new-privileges.

👥

Non-Root User

Runs as "sandbox" user (UID 1000). Never runs as root inside the container.

🔗

Network Isolation

Default: network=none. Members get bridge for API access. Guests get no network.

🔧

Tool Policy

Per-agent allow/deny lists. Guests blocked from write, edit, cron, gateway tools.

🕑

Auto-Prune

Idle containers pruned after 24h. Max age 7 days. Config hash mismatch triggers rebuild.

Sandbox Scopes

👤 Session Scope

One container per conversation session. Maximum isolation — each chat gets its own sandbox.

Container: openclaw-sbx-{session-slug}
Destroyed: after session archive (60min idle)

👥 Agent Scope

One container per agent ID. All sessions for the same agent share a container. Default for members/guests.

Container: openclaw-sbx-{agent-id}
Persistent: survives across sessions

🌐 Shared Scope

One container for all agents. Useful for service bots with shared workspace. Lowest isolation.

Container: openclaw-sbx-shared
Shared: all sessions see same filesystem
